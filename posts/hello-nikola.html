Firstly, I consider Wordpress for my blog. When I search on internet, I gets a lot of hate in the development community. Meanwhile, I didn't need a database and a gazillion server queries to post text on my website. So I started looking into static site generators. I considered:

  • JekyII + Octopress: Hooks directly into GitHub Pages and is apparently very good. But I love Python more, compare to Ruby.
  • Pelican: is a static site generator, written in Python, that requires no database or server-side logic.
  • Acrylamid: is a mixture of nanoc, Pyblosxom and Pelican licensed under BSD Style, 2 clauses.
  • hyde: is a static website generator written in python. While Hyde took life as awesome Jekyll's evil twin, it has since been completely consumed by the dark side and has an identity of its own.
  • Nikola

对比上述几个静态网站生成器，各有个的特点，很大程度上都可以满足我对建站的需求。

经过周末两天的调研，最终选择Nikola，主要是因为它支持多种标记语言的特点，可以让我在日后使用时多些选择，目前主要是用markdown和reStructuredText作为学习工作的记录方式。

Introduction

Nikola is a static site and blog generator, written in Python. It can use Mako and Jinja2 templates, and input in many popular markup formats, such as reStructuredText and Markdown and can even Jupyter (IPython) Notebooks into blog posts! It also supports image galleries, and is multilingual. Nikola is flexible, and page builds are extremely fast, courtesy of doit (which is rebuilding only what has been changed).

Features:

  • Host anywhere. Static websites are safer, use fewer resources, and avoid vendor and platform lock-in. You can host a Nikola website on any web server, big or small. It’s just a bunch of HTML files and assets.
  • Fast, incremental builds. Nikola is fast. We use doit, which provides incremental rebuilds — in other words, we rebuild only the pages that need rebuilding, saving CPU time, wall clock time and upload bandwidth.
  • Multiple input formats. Nikola will take input in many formats. Out of the box, we support reStructuredText, Markdown, IPython (Jupyter) Notebooks and HTML, and have plugins for many other formats.
  • Batteries included. Nikola comes with everything you need to build a modern website: blogs (with comments, tags, categories, archives, RSS/Atom feeds), multilingual support, easy image galleries, and code listings.
  • Easily extensible. Nikola is extensible. You can write a plugin to add any feature you want in a few lines of Python, or write your own theme in Mako or Jinja2. Or find something in the Plugin and Theme Indexes.
  • User-friendly CLI. Nikola has a friendly user interface that gets you up and running quickly and simplifies your work. You do not need to memorize headers just to create a post — we’ll write them for you.

Install

在Ubuntu14.04环境下的安装方式为:

$sudo aptitude install libxml2-dev libxslt1-dev python3-dev python3
$sudo pip3 install nikola requests webassets
$sudo pip3 install markdown
$sudo pip3 install --upgrade markdown

Initial blog

  • Initial nikola blog: $cd ~ && nikola init yellowsubmarine24.github.io
  • Modify publish time: Nikola默认new post的发布时间是推迟的，若想能build之后立即能看到post，则需要添加conf.py中的FUTRUE_IS_NOW = True。

Configure for markdown and reStructuredText

Nikola默认将后缀为.rst的文件视为reStructedText格式。若想使默认后缀为.md，使用语法更简洁的markdown，需要将conf.py中部分修改为：

POSTS = (("posts/*.md", "posts", "post.tmpl")
        ("posts/*.rst", "posts", "post.tmpl"),
        ("posts/*.txt", "posts", "post.tmpl"),)

Configure for github pages

$cd ~/yellowsubmarine24.github.io
$git init
$git branch src
$echo "# yellowsubmarine24.github.io" >> README.md
$nikola new_post
$git add conf.py README.md posts/* themes/*
$git commit -m "Inital blog and install 'zen' theme."
通过nikola build markdown的post，查看markdown语法和内容是否有错误或者不合理的地方
$nikola build
启动服务器
$nikola serve -a 127.0.0.1 -p 8080
通过chrome or firefox访问127.0.0.1:8080

为方便Nikola部署网站到github pages上, 需要修改的conf.py内容如下:

DEPLOY_COMMANDS = {
    'default': [
        "git checkout -B master",
        "rsync -rPv --delete-after --exclude .git --exclude .gitignore --exclude cache/ --exclude .doit.db.db output/ .",
        "git add -A",
        "git commit -a -m 'Updating blog content' ",
        "git push",
        "git checkout src",
    ]
}

GITHUB_SOURCE_BRANCH = 'src'
GITHUB_DEPLOY_BRANCH = 'master'
GITHUB_REMOTE_NAME = 'origin'

经过上述的配置，我们可以将其部署到github。首先需要申请github用户名，然后Create a repository，注意此仓库名必须与用户名一致，参见 github pages。

$cd ~/yellowsubmarine24.github.io
$git checkout master
$git remote add origin https://github.com/yellowsubmarine24/yellowsubmarine24.github.io.git
$git checkout src
$nikola github_deploy

注意在部署时一定要保证将index.html放在master分支下。
