    Firstly, I consider Wordpress for my blog. When I search on internet, I gets a lot of hate in the development community. Meanwhile, I didn't need a database and a gazillion server queries to post text on my website. So I started looking into static site generators. I considered:

    • JekyII + Octopress: Hooks directly into GitHub Pages and is apparently very good. But I love Python more, compare to Ruby.
    • Pelican: is a static site generator, written in Python, that requires no database or server-side logic.
    • Acrylamid: is a mixture of nanoc, Pyblosxom and Pelican licensed under BSD Style, 2 clauses.
    • hyde: is a static website generator written in python. While Hyde took life as awesome Jekyll's evil twin, it has since been completely consumed by the dark side and has an identity of its own.
    • Nikola

    对比上述几个静态网站生成器，各有个的特点，很大程度上都可以满足我对建站的需求。

    经过周末两天的调研，最终选择Nikola，主要是因为它支持多种标记语言的特点，可以让我在日后使用时多些选择，目前主要是用markdown和reStructuredText作为学习工作的记录方式。
